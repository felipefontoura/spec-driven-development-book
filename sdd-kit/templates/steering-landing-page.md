# Landing Page Steering

> Status: Draft
> Last Updated: YYYY-MM-DD

## Purpose

Reusable guidance for creating landing pages in this project.

## Standard Page Structure

1. Hero
2. Problem / Pain
3. Solution / Promise
4. Benefits
5. How It Works
6. Social Proof
7. Features or Use Cases
8. Pricing / Offer
9. FAQ
10. Final CTA

## Copywriting Rules

- Lead with user pain or desired outcome.
- Use clear, concrete language.
- Prefer short sections and scannable bullets.
- Every major section should support the primary CTA.
- Avoid vague claims without proof.

## Conversion Rules

- Primary CTA must be visible above the fold.
- Repeat CTA after proof and near the end.
- Reduce friction before asking for action.
- Address top objections in FAQ or proof sections.

## Design Rules

- Mobile-first layout.
- Strong visual hierarchy.
- Accessible contrast.
- Clear section rhythm.
- Avoid decorative complexity that weakens message clarity.

## Required Content Inputs

| Input | Description | Status |
|-------|-------------|--------|
| Audience | Who the LP is for | missing/known |
| Promise | Main transformation or benefit | missing/known |
| CTA | Desired action | missing/known |
| Proof | Testimonials, metrics, logos, examples | missing/known |
| Objections | User doubts to address | missing/known |

## Quality Checklist

- [ ] The hero explains who this is for and why it matters.
- [ ] The primary CTA is clear and repeated.
- [ ] Benefits are concrete, not generic.
- [ ] Proof supports the main promise.
- [ ] FAQ addresses real objections.
- [ ] Page works on mobile.
- [ ] Accessibility basics are covered.
